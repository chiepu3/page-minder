## 🚫 絶対に変えてはいけないロジック (Critical Logic protection)

### 1. SPA遷移検知とメモ再読み込み
- **現状**: `chrome.webNavigation` イベント（Background）と `MutationObserver` ベースの `useUrlWatcher`（Content Script）を併用しています。
- **理由**: Twitter (X) などのSPAでは、History APIのイベントだけではDOMの更新完了を検知できず、メモが表示されない問題が発生しました。
- **禁止事項**: `useUrlWatcher` を削除して `popstate` イベントのみに依存させること。これをすると多くのサイトでメモが表示されなくなります。

### 2. メモ更新と状態フラグの順序 (`MemoContainer.tsx`)
- **現状**: アクティブ化（Activation）の設定を行う際、**先に** `setReopenSettingsForMemoId` フラグを立ててから、`handleMemoUpdate` でメモデータを保存・更新しています。
- **理由**: 順序を逆にすると、メモ更新に伴う再レンダリング時にフィルタリングロジックが走り、アクティブ化有効なメモが一瞬アンマウントされて設定モーダルが強制終了してしまいます。
- **禁止事項**: `setReopenSettingsForMemoId` の呼び出し位置を `handleMemoUpdate` の後に移動すること。

### 3. Markdownライブラリの選定
- **現状**: `marked.js` を使用しています。
- **理由**: 以前使用していた `@uiw/react-md-editor` は、依存関係の問題で特定の環境下でUTF-8エンコーディングエラーを引き起こし、メモが表示されなくなる致命的なバグがありました。
- **禁止事項**: `@uiw/react-md-editor` やその派生ライブラリへの回帰。

## ⚠️ パフォーマンス最適化時の注意点 (Performance Pitfalls)

### 1. `useActivation` フックのイベントリスナー
- **現状**: `useRef` を使用してリスナーの再登録回数を最小限に抑えています。
- **注意**: `useEffect` の依存配列を安易に増やしてリスナーの着脱回数を増やすと、パフォーマンス劣化だけでなく、ホバー判定のチラつき（flickering）や、クリックイベントの予期せぬ多重発火を招きます。
- **最適化の方向性**: イベントデリゲーション（親要素での一括監視）への移行は有効かもしれませんが、`Shadow DOM` や `iframe` 内の要素への対応が複雑になるリスクがあります。

### 2. ストレージアクセス頻度
- **現状**: メモの位置情報や内容は変更のたびに `storage.local` に保存されます。
- **課題**: ドラッグ中の連続的な保存は負荷が高いです。
- **解決策案**: `lodash.debounce` 等導入による保存の遅延実行は推奨されますが、**タブを閉じた瞬間のデータロスト**を防ぐための対策（`beforeunload` でのフラッシュ等）が必須です。

### 3. アニメーションとレンダリング
- **現状**: `Memo` コンポーネントは多くのStateを持っています。
- **注意**: これらを `memo` 化 (`React.memo`) しても、結局 `props` (特にコールバック関数) が毎回再生成されていると意味がありません。
- **推奨**: `MemoContainer` 側で渡しているコールバック (`handleMemoUpdate` 等) が適切に `useCallback` されているか再確認し、子コンポーネントの不要なレンダリングを抑制してください。

### 4. ホバー猶予時間と一時停止ロジック (`useActivation.ts`)
- **現状**: カーソルがトリガー要素を離れても即座には非表示にならず、`activationHideGracePeriod` (デフォルト300ms) の猶予があります。この間にカーソルがメモ上に移動すれば、`pauseDeactivation` が呼ばれて非表示がキャンセルされます。
- **理由**: メモを読んだり編集したりするためにカーソルをメモ上に移動する時間を確保するため。
- **禁止事項**: 猶予時間のロジック (`graceTimeoutIdsRef`) や、メモ上ホバー時の一時停止ロジック (`ActivationOverlay` の `onMouseEnter`/`onMouseLeave`) を削除すること。

## ✅ 推奨される改善アプローチ

1. **バンドルサイズの削減**: 現在のビルドサイズを分析し、不要なライブラリや動的インポートできる部分を特定する。
2. **レンダリングの最適化**: React DevTools Profiler を使用して、ドラッグ時や入力時の無駄な再レンダリングを特定し、`React.memo` と `useCallback` / `useMemo` で適切に最適化する。
3. **イベントハンドリングの効率化**: 多数のメモが表示されている場合のイベントリスナーのオーバーヘッドを計測し、必要であれば最適化する。
