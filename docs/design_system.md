# PageMinder Design System & Accessibility Guidelines

このドキュメントでは、PageMinderのUI設計における色使い、コントラスト、およびテーマ適用の基準を定めます。

## 1. テーマの基本方針

PageMinderは、ユーザーの集中を妨げない「パステルカラー」と「シンプルさ」を基調とします。

### ライトテーマ (Light Theme)
- **原則**: 「紙」のような清潔感と可読性を重視。
- **背景**: 白（`#FFFFFF`）または非常に淡いグレー・パステルカラー。
- **文字**: 濃いグレー（`#333333`）またはテーマアクセント色の濃色（例：`#6D28D9`）。
- **注意点**: 背景がパステルカラーであっても、その上に乗せる文字はコントラスト比 4.5:1 以上を維持すること。

### ダークテーマ (Dark Theme)
- **原則**: 「真夜中」のような落ち着きと視認性を重視。
- **背景**: 深い紫（`#1A1A2E`）や濃い紺。完全な黒（`#000000`）は避け、奥行きを出す。
- **文字**: オフホワイト（`#E0E0E0`）や淡いパステルパープル。
- **アクセント**: 視認性の高い明るいパステルまたは彩度の高い色（例：`#B39DDB`）。

## 2. カラーパレットの選択基準

### 強調要素 (Primary Buttons / Icons)
| テーマ | 背景色 (BG) | 文字色 (Text/Icon) |
| :--- | :--- | :--- |
| **Light** | **淡色パステル** (例: `#F3E8FF`) | **濃色アクセント** (例: `#6D28D9`) |
| **Dark** | **鮮やかアクセント** (例: `#7C3AED`) | **濃色背景色** (例: `#1A1A2E`) |

> [!IMPORTANT]
> ライトテーマで鮮やかな背景色（例：濃い紫）を使用し、その上に黒文字を載せることは禁忌です。
> 背景を淡くして文字を濃くするか、背景を濃くして文字を白にするかの二択を徹底してください。

## 3. 実装のルール

- **CSS変数の活用**: `entrypoints/popup/style.css` などのグローバルスタイルでは、必ず `--popup-bg` などの変数を使用する。
- **インラインスタイルの例外**: Content Script（メモ本体）など、ホストページの介入を防ぐ必要がある箇所でのみインラインスタイルを許可する。その際も `THEMES` 定数から値を参照すること。
- **動的同期**: テーマの変更は `storage` を介して全タブに即座にブロードキャストすること。
